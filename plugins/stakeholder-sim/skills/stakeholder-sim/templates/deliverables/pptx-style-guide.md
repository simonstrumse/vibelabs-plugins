# PowerPoint deliverable style guide

An executive board deck generated by the `pptx-generator` agent must feel like it came out of McKinsey, Brunswick, or Teneo. This guide encodes the conventions that make executives trust a deck in five seconds.

## Canonical deck structure (12–16 slides)

A stakeholder reaction assessment deck, adapted for board consumption:

1. **Cover** — classification, title, client, date, prepared-by
2. **Agenda** — 5–7 section tiles
3. **Executive summary** — one slide, SCR structure (Situation / Complication / Resolution)
4. **The stimulus** — what's being assessed, in one slide
5. **Methodology** — agent pipeline + cohort composition, one slide
6. **Overall stakeholder posture** — reaction distribution, one slide with a table
7. **Consensus signal #1** — the single most important finding, full-slide pull quotes
8. **Consensus signal #2** — second convergence
9. **Consensus signal #3** — third (if applicable)
10. **Disagreement axes** — where stakeholders split, one slide
11. **Red-line warnings** — who will act publicly, one slide with a table
12. **Recommended actions** — priority-ranked table, one slide
13. **"So what"** — implications for the next 72 hours, one slide
14. **Coverage limitations** — honest about gaps
15. **Appendix cover** — section divider (optional)
16. **Appendix slides** — methodology detail, persona roster, raw reaction excerpts (optional, as needed)

For smaller runs (5-8 personas), collapse to 12 slides. For large runs (16-20 personas), extend appendix.

## Slide dimensions

16:9 widescreen, 13.33" × 7.5" (338.67mm × 190.5mm). Never 4:3.

## Color palette (HEX)

| Role | Hex |
|---|---|
| Primary — navy (title bg, section dividers, rule lines) | `#1C2B4A` |
| Content background | `#FFFFFF` |
| Accent — slate blue (highlights, pull-quote bars) | `#2E5FA3` |
| RAG — red (critical, hostile) | `#B91C1C` |
| RAG — amber (high, skeptical, monitor) | `#D97706` |
| RAG — green (stable, supportive) | `#2C6E35` |
| Text primary | `#1A1A1A` |
| Text secondary | `#6B7280` |
| Table row stripe | `#F0F2F5` |

No color outside this palette. No gradients. No drop shadows on text.

## Typography

| Element | Font | Size | Weight |
|---|---|---|---|
| Action title (top of every slide) | Georgia | 24pt | Bold |
| Section header | Calibri | 20pt | Bold |
| Body / bullets | Calibri | 14pt | Regular |
| Callout / pull quote | Georgia Italic | 18pt | Italic |
| Table header | Calibri | 11pt | Bold |
| Table body | Calibri | 11pt | Regular |
| Footer / classification | Calibri | 9pt | Regular |
| Source citation | Calibri | 8pt | Italic grey |

Georgia (serif) for action titles + pull quotes is the "serious consulting" pairing. Calibri is used everywhere else. No decorative fonts. Ever.

## Action title rule (non-negotiable)

Every slide's title must be a **complete sentence stating the conclusion**, active voice, specific, ≤15 words. A board member should be able to read only the titles and understand the argument ("ghost deck test").

**Weak → strong:**

- "Media coverage" → "Mongabay will lead with the 'terrorist attack' quote and CM 51-07 as the second paragraph"
- "Recommendations" → "Walk back 'terrorist attack' today — the single highest-impact action available at T+0"
- "Methodology" → "Eight modelled stakeholders reacted independently; three consensus signals emerge from cross-cluster convergences"

If a proposed title is not a complete sentence, rewrite it.

**For recommendation slides specifically:** use **verb-first imperatives** in the Bain convention. Every recommendation starts with an action verb: "Contain", "Walk back", "Acknowledge", "Engage", "Pre-wire". The verb IS the decision prompt. A noun phrase like "Same-day clarifying statement" is weaker than "Issue a same-day clarifying statement walking 'terrorist attack' back to 'reckless and dangerous'."

## Standard slide frame (every content slide)

| Element | Position | Spec |
|---|---|---|
| Action title | Top, full width | 24pt Georgia bold navy, left-aligned |
| Title accent rule | Below title | 2pt navy, 3" wide |
| Body content area | Center | Uses ~80% of slide area |
| Page number | Bottom-right | 9pt Calibri grey |
| Classification line | Bottom-center | 9pt Calibri grey, "Confidential — Client Leadership Only" |
| Document ref / prepared-by | Bottom-left | 9pt Calibri grey, "Vibe Labs · [run-ref]" |
| Source citation | Above footer, right-aligned | 8pt Calibri italic grey, if data cited |

## Cover slide

Full navy background. White type only.

- Reference number at top: 10pt Calibri 0.2em tracked, uppercase
- Title: Georgia 40pt light white — e.g. "Stakeholder Reaction Assessment"
- Subtitle: Georgia italic 20pt, slate blue — e.g. "Bandero incident response statement"
- Thin accent rule: 2pt slate blue, 2" wide
- Client block (labels in Calibri 9pt uppercase tracked, values in Calibri 12pt):
  - CLIENT · Aker BioMarine ASA & Aker QRILL Company
  - PREPARED FOR · [named executives]
  - PREPARED BY · Stakeholder Simulation Practice, Vibe Labs
  - ASSESSMENT DATE · [date]
- Classification footer band: red `#B91C1C`, 1pt top/bottom borders, 9pt uppercase tracked white text

## Executive summary slide (SCR structure)

One slide, three bold sentences (situation/complication/resolution), each with 1-2 supporting bullets.

```
ACTION TITLE: [The single most important finding as a sentence]
────────────
SITUATION         [One-line context]
• [supporting bullet]

COMPLICATION      [One bold claim]
• [supporting bullet]

RESOLUTION        [One bold claim — recommended action]
• [supporting bullet 1]
• [supporting bullet 2]
────────────
DECISION REQUIRED: [specific ask, in a red-bordered box at bottom]
```

## Pull quote slide

Used for consensus signal slides. Up to 3 direct quotes, each with:
- Opening `"` mark at 60pt in slate blue (decorative, can be a shape)
- Quote body in Georgia italic 16pt dark grey
- 4pt accent bar on the left in slate blue
- Attribution on a new line in Calibri 11pt bold slate blue: `Name · Institution`

## RAG indicators

A 12pt solid circle in the corresponding hex. **Always accompanied by a text label.** Never color alone. Use `add_shape(MSO_SHAPE.OVAL, ...)` in python-pptx.

## Recommendation table slide

Columns: Priority # / Action / Owner / Timing / Status (RAG). Five rows max. Header row: navy fill, white 11pt bold. Body rows: alternating white and `#F0F2F5`. No cell borders within rows; 1pt navy bottom-border only on header.

## Tables generally

- No visible inner cell borders
- Header row: navy fill `#1C2B4A`, white text 11pt bold, centered
- Body rows: alternating `#FFFFFF` and `#F0F2F5`
- Narrative column gets 60-70% of width
- Full slide width minus margin

## "So what" slide

3 numbered "therefore" statements, each 1 sentence, each leading to a specific recommendation. No chart. No bullets. Just three sentences laid out vertically with spacing.

## Appendix dividers

Optional navy full-slide dividers between major appendix sections.

## What NOT to include

- No stock photography
- No clipart or decorative icons
- No company/firm logos beyond a small wordmark in the footer
- No transitions, animations, or builds
- No slide counter on the cover
- No "Questions?" slide — end on recommendations
- No emoji
- No color outside the palette

## python-pptx workflow

Use a pre-built master `.pptx` template. If one doesn't exist in `templates/deliverables/master.pptx`, the generator agent creates it programmatically on first run using the slide_width / slide_height / master-slide-with-placeholders pattern.

Open template:

```python
from pptx import Presentation
prs = Presentation("templates/deliverables/master.pptx")
```

Add a content slide with action title:

```python
def add_action_title(slide, text):
    tx = slide.shapes.add_textbox(Inches(0.5), Inches(0.35), Inches(12.33), Inches(1.0))
    tf = tx.text_frame
    tf.word_wrap = True
    p = tf.paragraphs[0]
    p.text = text
    p.font.name = "Georgia"
    p.font.size = Pt(24)
    p.font.bold = True
    p.font.color.rgb = RGBColor(0x1C, 0x2B, 0x4A)
    return tx
```

Full working patterns are in the research at `research/deliverable-conventions/executive-deck-design.md` — the pptx-generator agent may use these verbatim.

## Rendering

```bash
python3 generate_deck.py  # writes deliverable.pptx
```

Optional PDF export for preview (agent not required to do this, but nice):

```bash
soffice --headless --convert-to pdf deliverable.pptx
```

## The deliverable the agent produces

The PPTX file opens directly in PowerPoint (all versions 2016+), Keynote, and Google Slides. The executive opens it, skims the action titles (ghost deck test), then spends time on the executive summary and recommendations. Every claim has a named stakeholder behind it. Every recommendation has a priority and a timeline. The deck circulates among the leadership team in hours and gets annotated in place. That's what the deliverable must support.
